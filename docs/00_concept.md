# Analysis Engine Library（分析エンジンライブラリ）設計思想

## 1. このプロジェクトの目的

このプロジェクトは、ゲームプレイから得られる行動データをもとに、性格・認知・思考・行動傾向・プレイスタイルを分析するための共通基盤を作ることを目的とする。

最終目的は、複数のアプリやゲームで共通利用できる分析エンジンを構築し、エンターテインメント性と差別化の高いアプリ群を展開することである。

## 2. 基本方針

この分析エンジンは医療診断を行うものではない。

ただし、医学・心理学・認知科学・神経心理学・教育心理学・行動科学などで使われている評価軸は参考にする。

目的は、診断の代替ではなく、ゲーム体験を通じて「自分でも気づきにくい傾向」を楽しく可視化することである。

## 3. 重視する価値

- エンターテインメントとして面白いこと
- ユーザーが納得感を持てること
- 医学・心理学的な知見を可能な範囲で尊重すること
- 断定表現を避け、傾向として表現すること
- 複数アプリに再利用できること
- データ容量と作業効率を意識すること

## 4. このエンジンで扱うもの

主に以下を扱う。

- 性格特性
- 認知能力
- 注意
- 記憶
- 実行機能
- 判断・意思決定
- 学習傾向
- 感情反応
- 動機
- 社会性
- 行動傾向
- ストレス下の変化
- ゲーム固有のプレイスタイル

## 5. このエンジンで扱わないもの

以下は扱わない、または慎重に扱う。

- 医療診断
- 疾患の判定
- 治療方針の提示
- 知能や人格の断定
- ユーザーを不安にさせる表現
- 差別や偏見につながる分類

## 6. 辞典方式を採用する理由

分析項目、検査、ゲーム、診断文、関連情報を分けて管理する。

同じ説明を複数箇所に保存せず、IDで参照することで、容量を抑え、修正を一箇所で済ませる。

この方式により、後からゲームや診断文が増えても、土台を壊さず拡張できる。
